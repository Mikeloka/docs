14 ### Comments
15 In Aurora code files we have a very well defined anatomy for comment blocks. You have three types of comments: the initial comments, documentation comments and code comments.
16 
17 #### Initial Comments
18 Initial comments have general information about the file and should be present on ALL source code files. We have a boilerplate for C-like languages and languages that have `#` as 'keyword' for comment blocks. Depending on the license of the repository, you should use GPL-3.0 and LGPL-3.0 boilerplates. You can found it [here](BOILERPLATES.md).

21 #### Documentation Comments
22 To compile our documentation we use Doxygen, which uses JavaDoc Style for C-like languages and Doxygen style for other languages as a comment standard for automatically code structure detection. When you write a code file you should use this structure on header files:
23 ```cpp
24 /** Example comment
25  * Another comment
26  *
27  */
28 ```
29 or this, in case of a non C-like language:
30 ```cmake
31 ## Example comment
32 # Another comment
33 #
34 ```
35 
36 You should use special commands to Doxygen know what you want to do with this comment. Here is some useful and most used commands:
37 - `@file`
38 - `@param`
39 - `@return`
40 - `@see`
41 - `@author`
42 
43 You can find the full list [here](http://www.stack.nl/~dimitri/doxygen/manual/commands.html).
44 
45 For better codestyle we activated the `JAVADOC_AUTOBRIEF`, which means that you can automatically initialize a brief description ending with a first dot followed by a space or a new line. For example:
46 ```cpp
47 /** Brief description which ends at this dot. Details follow
48  * here.
49  */
50 ```
51 **Note:** Because of the aesthetics, use the brief in the first line (follow the example) whenever possible.
52 
53 Here is a practical example of a documentation comment:
54 ```cpp
55 /** Constructs a vector with the given coordinates.
56  * @param x The x value for the x coordinate.
57  * @param y The y value for the y coordinate.
58  * @see Vector2D( )
59  * @see Vector2D(float )
60  */
61 ```
